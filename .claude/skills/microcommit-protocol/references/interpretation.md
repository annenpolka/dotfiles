# ユーザー発話の解釈ルール

自然言語をコマンドとして解釈する。ユーザーはコマンドを覚える必要はない。

## 承認

| 発話例 | 動作 |
|--------|------|
| OK / k / おk | 承認、実装して次へ |
| うん / はい / いいよ | 同上 |
| 👍 / yes / go | 同上 |
| next / 次 | 同上 |

## 詳細要求

| 発話例 | 動作 |
|--------|------|
| ? / 詳しく / detail | 現在のブロックを詳細展開 |
| なんで / why / 理由は | なぜこのブロックが必要か説明 |
| どういうこと / what | 何をしているか再説明 |

## 粒度変更

| 発話例 | 動作 |
|--------|------|
| もっとまとめて / 大きく見せて | Level を1つ上げる |
| 細かく見せて / もっと詳しく分けて | Level を1つ下げる |
| level 0 / level 1 / level 2 | 指定レベルに変更 |

## ナビゲーション

| 発話例 | 動作 |
|--------|------|
| 3番に戻って / back 3 / #3 | 指定ブロックを再提示 |
| 飛ばして / skip / 後で | 保留して次へ（後で戻る） |
| 今どこ / status / 進捗 | 完了・保留・残りを表示 |

## 修正

| 発話例 | 動作 |
|--------|------|
| 〜を追加して | 指定内容を追加して再提示 |
| 〜じゃなくて〜 | 指定内容を変更して再提示 |
| 〜は要らない | 指定内容を削除して再提示 |

## 質問

| 発話例 | 動作 |
|--------|------|
| 〜ってどういう意味？ | 用語・概念を説明 |
| 〜と〜の違いは？ | 比較説明 |
| 他の選択肢は？ | 代替案を提示（選択はユーザー） |

## 解釈の優先順位

1. 明示的なコマンド（`OK`, `level 2` など）
2. 肯定/否定の判定
3. 疑問文かどうか
4. 修正要求として解釈

曖昧な場合は確認する。勝手に解釈して進めない。
